Troubleshooting

Frequent Login Issues (Private/Randomized WiFi Address)

iPhone:

  1. Go to Settings > Wi-Fi.
  2. Tap the info () button next to ShipWide.
  3. Turn off Private Address.
  4. Restart WiFi.

Android:

  1. Go to Settings > Connections > Wi-Fi.
  2. Tap the gear icon next to ShipWide.
  3. Tap View More > MAC Address Type..
  4. Select Phone MAC to change the MAC address type from Randomized MAC to Phone MAC..
  5. Restart WiFi.

Tip: When you leave the ship, its best to switch back to Private/Randomized MAC.

Taani has local file shares:

xx \\data\cruisedata Read-only underway data

Mapping a Shared Drive

Use your username and password to map a shared drive. If you lack access, contact science support.

Folder Structure

  • s2s - Files synchronized between Ship and Shore.
  • shipside - Files stored only on board.
  • temp - Temporary files.

Connection Instructions

Connecting Windows Clients
  1. Right-click This PC in File Explorer, then select Map network drive.
  2. Select a drive letter and enter a location from the table above.
  3. Check "Reconnect at sign-in" and "Connect using different credentials".
  4. For OpenVDM, use username "guest" (no password).
  5. For other shares, use "username" and enter your password.
  6. Check "Remember my credentials".
Connecting Mac Clients
  1. In Finder, select Go → Connect to Server.
  2. Enter the server address (e.g., smb://files/science).
  3. Click + to save to favorites.
  4. Select "Registered User" and enter credentials as username or "guest".
  5. Check "Remember this password in my keychain".

Scanning Documents

Scan documents using the Color Printer in the Computer Lab.

  1. Wake printer with the "Start" button or tap touch screen.
  2. Select "Send" on the touch screen.
  3. Choose "Address Book" → "Scan" → "OK".
  4. Place document in feeder & press "Start".
  5. Download your scanned document here.
Mapping the Scans Shared Drive

Windows

Go to Start → Map Network Drive and enter:

Mac

In Finder, go to Go → Connect to Server and enter:

smb://files.sikuliaq.alaska.edu/Scans/

Linux

In File Manager, select Connect to Server and enter:

WebDAV (HTTP) → files.sikuliaq.alaska.edu (Port 80) → /Scans

Network Acceptable Use & Cybersecurity

Learn about the rules, monitoring, and security practices for using the R/V Taani’s onboard network and internet resources.

Acceptable Use Policies

The R/V Taani is owned by the National Science Foundation (NSF) and operated by the Oregon State Universty. Use of network resources is subject to policies from NSF, UAF, and OSU.

Oregon State Universty Policies:

  • Acceptable Use Policy
  • Partisan Political Activity

Acceptable Use: Network resources onboard are primarily for science and ship operations. Personal use is permitted as long as it doesn’t interfere with mission-critical tasks.

Monitoring: Network activity is monitored to manage limited bandwidth and ensure efficient usage.

By using network resources aboard Sikuliaq, you agree to these terms of use.

Taani Cybersecurity

Please stay alert and follow good cybersecurity practices. You can review the Taani Cybersecurity Awareness Training slide deck for guidance.
